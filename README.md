# SAIDIO Operations Desk

Static GitHub Pages dashboard and **daily prompt factory** for creative assets across three
streams — music, voiceover and Suntravel visuals. The site is a unified, taggable, calendar-based
library of every generated prompt; the media itself is generated by hand from these prompts and
stored locally under `resource/`.

## Streams & schedule (all use the same two secrets, text-only, free tier)

| Workflow | Cron (UTC / JST) | Output | Discord |
|---|---|---|---|
| `daily-brief.yml` (music) | 00:12 / 09:12 | `data/dashboard.json` — 10 music prompts | 🎵 |
| `daily-voiceover.yml` | 00:18 / 09:18 | `data/voiceover.json` — 4 narration scripts + 2 SFX prompts | 🗣️ |
| `daily-suntravel.yml` | 00:24 / 09:24 | `data/suntravel.json` — 6 B-roll + 1 Flow + 3 content + 2 card prompts | 🎬 |

Voice/character consistency is governed by [`VOICES.md`](VOICES.md) — the single source of truth
for every narration voice and drama character. Lock a voice name there once, reuse it forever.

## Local preview

Open `index.html`, or use any static file server. No build step is required.

## Daily automation setup

Create repository secrets before enabling `.github/workflows/daily-brief.yml`:

- `GEMINI_API_KEY`: Gemini API key used to generate the brief and prompt variants.
- `DISCORD_WEBHOOK_URL`: Discord incoming webhook. Never place it in a tracked file.

The workflow updates `data/dashboard.json`, commits the day’s archive and POSTs the same formatted message to Discord. GitHub Pages deploys when `main` changes.

## Data contract

`data/dashboard.json` is the public source used by the website. Do not include private keys, webhook URLs, client data or unpublished performance data in it.
